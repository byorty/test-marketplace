# Маркет плейс

## Контекст

Требуется реализовать два микросервиса:
1. Микросервис товаров.
2. Микросервис заказов.

## Микросервис товаров

Микросервис должен иметь функциональность:
1. Просмотр списока товаров с фильтром по названию, цене, сроку доставки, категории, рейтингу. Предусмотреть пагинацию, сортировку по рейтингу и цене.
2. Просмотр конкретный товар по идентификатору.
3. Создание, изменение, удаление товара.

Методы просмотра доступны и покупателю и сотруднику.
Методы создания, изменения, удаления доступны только сотруднику.

## Микросервис заказов

Микросервис должен иметь функциональность:
1. Добавление товара в корзину.
2. Оформление заказа.
3. Просмотр заказа: идентификатор, статус, стоимость, дата создания, дата доставки, перечень товаров.

Все методы добавления товаров и фофрмления заказа доступны покупателю.
Метод просмотра доступен и покупателю и сотруднику.

## Задачи
1. Согласовать модель данных товара и заказа.
2. Согласовать API.
3. Согласовать общую архитектуру микросервиса(-ов).
4. Наладить генерацию кода из Swagger.
5. Наладить миграцию структуры БД.
6. Реализовать микросервис товаров.
7. Покрыть микросервис товаров unit-тестами.
8. Реализовать микросервис заказов.
9. Покрыть микросервис заказов unit-тестами.
10. Упаковать микросервис в образ.
11. Подготовить docker-compose, в котором будет подниматься БД, отрабатывать миграции БД, запускаться микросервисы.
